# 2026-03-01 目录结构清爽化复盘

## 结论

项目核心代码结构不算屎山，问题主要是工作区外层混杂导致维护成本上升：

- 代码主干：`app`（前端）和 `backend`（后端）边界清晰
- 外层问题：临时工程、备份、旧站、重复 docs、过程笔记混放
- 最大风险：同一份研究数据有双份副本，容易改错位置

## 本次整理动作

1. 根目录只保留核心入口：`app/`、`backend/`、`docs/`、`notes/`、`archive/`
2. 根目录重复 `docs` 已归档，`app/docs` 明确为唯一主源
3. 临时工程、旧站、备份统一归档到 `archive/`
4. 脑暴和维护日志归并到 `notes/`
5. 清理 `app/src` 中未使用的 Vite 模板遗留文件

## 结构债与建议

### P0（立刻做）

- 做 `app/docs/*.csv -> app/src/data/countries.json` 自动生成脚本
- 在 CI 增加数据一致性校验，防止手工同步偏差

### P1（短期）

- 根目录补统一任务入口（如 `Makefile` / `justfile`）
- 约定归档规范：`temp-*`、备份包、外部导出的 HTML 统一进 `archive/`

### P2（中期）

- 决策仓库形态：维持“`app` 独立仓库 + workspace”，或改造为真正 monorepo

## 判断标准（后续验收）

- 新成员 5 分钟内能判断“哪里是生产代码、哪里是历史资料”
- 任意文档字段能追溯到唯一主源
- 每次数据变更都能自动更新前端数据并通过构建
