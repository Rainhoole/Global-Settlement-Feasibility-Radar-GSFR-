# GSFR 前端视觉改版（Figma 执行任务书）

## 1. 项目目标

在 **不改变现有功能、信息架构、数据逻辑** 的前提下，完成一次纯视觉层改版。  
新视觉语言：**Generative Grid / Pixel Map + Halftone + Single Accent Color**。

一句话定义：  
**用网格、半调和单一强调色，做抽象的自画像。**

## 2. 改版边界（必须遵守）

- 不改功能：地图、筛选、搜索、表格视图、详情侧栏保持现状。
- 不改交互路径：用户完成任务的步骤不增加。
- 不改数据结构：接口字段、前端数据模型、筛选逻辑不变。
- 可用性不降级：桌面端与移动端都可完整使用。

## 3. 视觉方向（必须体现）

### 3.1 Generative Grid / Pixel Map

- 使用小方块/点阵构成抽象图形层。
- 图形应有“热力图 + 雷达图”的信息感，不做纯装饰。
- 可作为背景层、分区层、数据卡片底纹。

### 3.2 Halftone

- 在 Hero 区或详情头图区引入半调网点化轮廓。
- 质感接近报纸印刷放大颗粒，避免照片写实感。
- 半调层透明度与密度可参数化，便于后续主题扩展。

### 3.3 Single Accent Color System

- 一套主题仅一个强调色，其他全部中性色。
- 首版至少交付 3 套主题：
  - Theme A：Orange
  - Theme B：Deep Green
  - Theme C：Monochrome (Black/White)

## 4. Design Tokens（需在 Figma Variables 交付）

最少包含以下变量组：

- `color.bg`
- `color.surface`
- `color.text.primary`
- `color.text.secondary`
- `color.accent`
- `color.border`
- `color.grid.dot`
- `effect.halftone.opacity`
- `effect.halftone.scale`
- `radius.sm/md/lg`
- `space.4/8/12/16/24/32`
- `font.display`
- `font.body`
- `font.mono`
- `motion.fast/base/slow`

约束：

- 对比度满足 WCAG AA（正文至少 4.5:1）。
- 颜色数量控制：每主题最多 1 个强调色 + 中性色阶。

## 6. 组件交付清单（含状态）

以下组件必须有 `default / hover / active / disabled`：

- Button（Primary / Secondary / Ghost）
- Input（含搜索态）
- Select / Filter Chip
- Tabs（地图/表格切换）
- Table Row / Table Header
- Tooltip
- Legend Item
- Side Panel Section
- Modal

地图相关补充：

- Country 状态着色规范（active / just_launched / coming_soon / not_covered）
- Hover 与 Selected 的层级与描边规则

## 7. 动效规范（轻量）

- 页面进入：网格层渐显（200-400ms）
- 筛选切换：卡片/列表短过渡（150-250ms）
- 禁止复杂炫技动画，优先信息稳定性与响应速度

- 设计 token 足够工程直接落地
