# Global Settlement Feasibility Radar — 前端设计说明书

> 本文档供 UI/UX 设计师使用，包含完整的页面结构、交互逻辑、视觉规范和组件定义。

---

## 一、产品概述

### 1.1 产品名称
**Global Settlement Feasibility Radar**（全球航运即时结算可行性雷达）

### 1.2 一句话描述
一个以世界地图为核心界面的情报系统，帮助用户在 3 分钟内判断某个国家是否理论上支持跨境即时结算（稳定币/外汇/银行通道），所有判断均可追溯至官方来源。

### 1.3 目标用户
- 航运公司战略/财务团队
- 港口运营公司
- 跨境结算网络的 BD 团队
- 稳定币支付基础设施决策者
- 投资人（演示用途）

### 1.4 核心场景
| 场景 | 用户行为 | 系统反馈 |
|------|---------|---------|
| 全局一览 | 打开网页 | 看到着色的世界地图，一眼识别机会分布 |
| 快速判断 | 鼠标悬停某国 | 弹出浮窗，显示国名+评分+状态 |
| 深度研究 | 点击某国 | 右侧滑出详情面板，展示完整合规数据和来源 |
| 筛选聚焦 | 使用顶部筛选器 | 地图上只保留符合条件的国家着色 |
| 对比分析 | 切换到列表视图 | 所有国家按评分排列的表格，可排序 |
| 走廊评估 | 选择出发国→目的国 | 显示两国对比 + 走廊可行性 + 地图连线 |

### 2.1 设计调性
**"作战室 / 情报雷达"** — 深色背景、数据密集、专业感强，类似：
- Bloomberg Terminal 的数据密度
- 卫星地图控制台的科技感
- 军事态势图的信息分层

| 筛选器 | 选项 |
|--------|------|

当前激活的视图按钮高亮（`#3b82f6` 底色）。

### 4.2 世界地图 (Map View) — 核心视图

#### 地图基础
- 投影: Mercator（用户更熟悉的商用地图投影）
- 底图颜色: 海洋 `#0a0f1e`（与页面背景融合），国界线 `#1e293b`
- 国家填充: 按可行性指数着色（见色阶表）
- 未评估国家: `#1e293b`（比海洋稍亮，可见但不抢眼）

#### 交互行为

| 操作 | 反馈 |
|------|------|
| 鼠标悬停国家 | 该国边界高亮（白色描边 2px），显示 Tooltip |
| 点击国家 | 该国边界变为蓝色描边（`#3b82f6` 3px），右侧弹出详情面板 |
| 滚轮缩放 | 地图平滑缩放（最小 1x，最大 8x） |
| 拖拽 | 地图平移 |
| 双击 | 放大到该区域 |
| 点击空白处 | 关闭详情面板，取消选中 |

#### 地图装饰元素
- **网格线:** 经纬线以极低透明度显示（`#1e293b` 30% opacity），增强科技感
- **区域标签:** 主要区域名（MENA, SEA, LATAM...）以小号半透明文字标注在对应位置

#### 地图上的走廊连线
选择两国后，在地图上画一条弧线连接两国（大圆弧线），颜色反映走廊可行性：
- 绿色弧线 = 两端都可行
- 黄色弧线 = 某端有限制
- 红色弧线 = 某端不可行

弧线带轻微动画（流动光点从 From 到 To），强化"资金流向"感。

## 七、空状态与边界情况

| 情况 | 设计 |
|------|------|
| 首次加载 | 地图居中显示全球，不选中任何国家，底部提示 "点击任意国家查看详细评估" |
| 筛选无结果 | 所有国家变灰，地图中心显示 "没有符合当前筛选条件的国家" |
| 数据不足的国家 | 灰色填充，Tooltip 显示 "⚠️ 该国尚未完成评估" |
| 某字段数据缺失 | 字段值显示 "— 未确认"（灰色斜体），不参与评分计算 |
| 走廊查询同一国家 | 提示 "请选择不同的出发国和目的国" |
| 网络加载地图数据中 | 地图区域显示骨架屏 (skeleton) + "加载地图数据..." |
